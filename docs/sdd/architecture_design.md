# CScriptEngine 架构设计文档

## 1. 项目概述

CScriptEngine 是一个轻量级的 C 语言脚本引擎，能够解析和执行 C 语言风格的脚本代码。它通过词法分析、语法分析、语义分析、中间代码生成、目标代码生成和执行六个主要阶段来处理脚本文件。

### 1.1 主要功能

#### 1.1.1 数据类型
- **基本类型**: int、double、char、bool、float、void - ✅ 全部支持
- **构造类型**: 数组 - ✅ 支持
- **指针类型**: 指针 - ✅ 支持
- **结构体类型**: struct - ✅ 支持
- **联合体类型**: union - ⚠️ 部分支持
- **枚举类型**: enum - ✅ 支持
- **类型别名**: typedef - ✅ 支持
- **字符串类型**: 字符串字面量 - ✅ 支持

#### 1.1.2 运算符
- **算术运算符**: + (加)、- (减)、* (乘)、/ (除)、% (取模) - ✅ 全部支持
- **关系运算符**: ==、!=、<、<=、>、>= - ✅ 全部支持
- **逻辑运算符**: && (逻辑与)、|| (逻辑或)、! (逻辑非) - ✅ 支持
- **位运算符**: & (按位与)、| (按位或)、^ (按位异或)、~ (按位取反)、<< (左移)、>> (右移) - ✅ 全部支持
- **赋值运算符**: =、+=、-=、*=、/=、%= - ✅ 全部支持
- **自增/自减**: ++、-- - ✅ 支持
- **条件运算符**: ?: - ✅ 支持
- **取地址/解引用**: &, * - ✅ 支持
- **数组下标**: [] - ✅ 支持
- **成员访问**: . (点运算符) - ✅ 支持

#### 1.1.3 控制流语句
- if-else 条件语句 - ✅ 支持
- while 循环 - ✅ 支持
- for 循环 - ✅ 支持
- do-while 循环 - ✅ 支持
- switch 多分支 - ✅ 支持
- break 语句 - ✅ 支持
- continue 语句 - ✅ 支持
- goto 语句 - ⚠️ 部分支持
- return 语句 - ✅ 支持

#### 1.1.4 函数
- 函数定义和声明 - ✅ 支持
- 函数调用 - ✅ 支持
- 参数传递 - ✅ 支持
- 返回值 - ✅ 支持
- 递归调用 - ✅ 支持
- extern 外部声明 - ✅ 支持
- 内联汇编 - （暂不支持）

#### 1.1.5 变量
- 变量声明 - ✅ 支持
- 变量赋值 - ✅ 支持
- 全局变量 - ✅ 支持
- 局部变量 - ✅ 支持
- 常量 (const) - （暂不支持）
- 静态变量 (static) - （暂不支持）

#### 1.1.6 其他特性
- 注释 (// 和 /* */) - ✅ 支持
- 预处理指令 (#define等) - （暂不支持）
- 标准库函数 (printf, puts等) - ✅ 部分支持
- 字符串字面量 - ✅ 支持

### 1.2 技术栈

- **C++ 17**
- **ANTLR4** 用于词法和语法分析
- **x86-64 机器码生成**（JIT）
- **x86-32 机器码生成**（JIT）
- **RISC-V 64位机器码生成**（JIT）
- **Windows 平台**（使用 VirtualAlloc 分配可执行内存）
- **RISC-V 平台**（K210）

### 2.3 核心模块

#### 2.3.1 前端 (Frontend) - 词法分析和语法分析

- **词法分析器（ANTLR4）**：将脚本文件中的字符流转换为词法单元（tokens）
- **语法分析器（ANTLR4）**：将词法单元流转换为抽象语法树（AST）
- **AST节点**：每个节点继承自基类Node，实现codeGen方法

#### 2.3.2 语义分析器 (SemanticAnalyzer)

- **符号表管理**：支持嵌套作用域、符号查找、作用域链遍历
- **类型检查**：类型兼容性验证、类型推断
- **错误报告**：语义错误检测和报告

#### 2.3.3 中间表示 (IR)

- **四元式生成**：将AST转换为三地址码
- **临时变量管理**：自动生成临时变量名
- **控制流翻译**：if/while/for语句的标签生成

#### 2.3.4 后端 (Backend)

- **目标代码生成**：将四元式转换为x64、x86-32或RISC-V 64位机器码
- **JIT执行**：分配可执行内存并执行生成的代码

#### 2.3.5 公共组件 (Common)

- 类型定义、值定义、内存分配指令等

## 4. 核心组件详细设计

### 4.1 前端 - 词法分析器

- **文件**：`src/antlr4/tokens.g4`（源文件）、`src/antlr4/tokens.cpp`（生成文件）
- **功能**：定义词法规则，将输入字符流转换为词法单元
- **主要词法单元**：
  - 关键字：int、double、char、bool、void、if、else、while、for、do、return、extern
  - 运算符：+、-、*、/、==、!=、<、<=、>、>=、=、&、;
  - 标识符：变量名、函数名
  - 常量：整数常量、浮点数常量、字符串常量
  - 标点符号：(、)、{、}、[、]、.、,

### 4.2 语法分析器

- **文件**：`src/antlr4/parser.g4`
- **功能**：定义语法规则，构建抽象语法树
- **主要语法规则**：
  - 程序结构：函数声明、变量声明、语句块
  - 表达式：二元表达式、一元表达式、函数调用、数组访问
  - 语句：赋值语句、返回语句、条件语句、循环语句

### 4.3 抽象语法树（AST）

- **文件**：`src/antlr4/node.h`
- **功能**：定义 AST 节点类型，表示程序的语法结构
- **主要节点类型**：
  - NBlock：语句块
  - NFunctionDeclaration：函数声明
  - NVariableDeclaration：变量声明
  - NAssignment：赋值语句
  - NReturnStatement：返回语句
  - NBinaryOperator：二元运算符
  - NIdentifier：标识符
  - NInteger：整数常量
  - NDouble：浮点数常量

### 4.4 语义分析器

- **文件**：`src/SemanticAnalyzer.h`、`src/SemanticAnalyzer.cpp`
- **功能**：进行语义分析，包括符号表管理、类型检查、作用域分析

### 4.5 中间代码生成器

- **文件**：`src/IRGenerator.h`、`src/IRGenerator.cpp`
- **功能**：将 AST 转换为中间代码（四元式）
- **四元式结构**：
  - `OpEnum`：操作枚举（ADD、DEL、MUL、DIV、EQ、JMP、JEZ、JNZ、CALL 等）
  - 每个四元式包含操作码、两个操作数、一个结果

### 4.6 目标代码生成器

- **文件**：`src/X64Code.h`、`src/X64Code.cpp`、`src/RV64Code.h`、`src/RV64Code.cpp`
- **功能**：将四元式转换为x86-64或RISC-V 64位汇编代码
- **全局变量支持**：
  - 使用数据段（.data section）存储全局变量
  - 使用 RIP-relative 寻址（x64）或绝对寻址（RV64）访问全局变量

### 4.7 代码执行器 (JIT)

- **文件**：`src/Code.h`、`src/Code.cpp`
- **功能**：实现JIT（即时编译）执行，分配可执行内存并执行生成的代码
- **平台适配**：
  - Windows：使用 VirtualAlloc
  - RISC-V：使用 mmap 或静态内存分配

### 4.7.1 JIT执行原理

JIT（Just-In-Time）编译是一种将源代码或中间代码在运行时转换为机器码并立即执行的技术。在CScriptEngine中，JIT执行的核心步骤包括：

1. **内存分配**：使用平台特定的API分配可执行内存
2. **内存保护**：设置内存保护属性为可执行
3. **代码复制**：将生成的机器码复制到分配的内存中
4. **函数转换**：将内存地址转换为函数指针
5. **代码执行**：调用函数指针执行代码
6. **内存释放**：释放分配的内存

### 4.7.2 JIT性能优势

- **执行速度快**：直接执行机器码，避免了解释执行的开销
- **内存使用高效**：只在需要时编译和执行代码
- **动态优化**：可以根据运行时信息进行优化
- **跨平台支持**：可以为不同平台生成相应的机器码

### 4.8 错误处理模块

- **文件**：`src/frontend/semantic/SemanticAnalyzer.cpp`、错误报告相关函数
- **功能**：提供编译时和运行时的错误检测与报告
- **主要功能**：
  - **词法错误**：无法识别的字符、无效的数字格式
  - **语法错误**：语法规则不匹配、缺少分号或括号
  - **语义错误**：未定义变量、未定义函数、类型不匹配、参数数量不匹配
  - **运行时错误**：除零错误、数组越界（待实现）
- **错误报告机制**：
  - 报告错误位置（文件名、行号、列号）
  - 提供可读的错误消息
  - 支持错误恢复，继续检测更多错误

### 4.9 符号表管理

- **文件**：`src/frontend/semantic/SemanticAnalyzer.h` 中的符号表类
- **功能**：管理程序中的所有符号（变量、函数、类型）
- **主要功能**：
  - **作用域管理**：支持全局作用域和局部作用域（函数体）
  - **符号存储**：存储变量名、类型、作用域、内存偏移量
  - **符号查找**：按名称查找符号，支持作用域链查找
  - **类型信息**：记录变量的数据类型（int、double、char、bool等）

### 4.7.3 实现细节

```cpp
// JIT执行核心代码
void* Code::exec() {
    // 分配可执行内存
    void* memory = VirtualAlloc(
        nullptr,
        m_CodeBuffer.size(),
        MEM_COMMIT | MEM_RESERVE,
        PAGE_EXECUTE_READWRITE
    );
    
    // 复制代码到内存
    memcpy(memory, m_CodeBuffer.data(), m_CodeBuffer.size());
    
    // 返回函数指针
    return memory;
}

// 执行代码
int Code::run() {
    // 获取可执行内存
    void* execMemory = exec();
    
    // 转换为函数指针
    typedef int (*FunctionType)();
    FunctionType function = (FunctionType)execMemory;
    
    // 执行代码
    int result = function();
    
    // 释放内存
    VirtualFree(execMemory, 0, MEM_RELEASE);
    
    return result;
}
```

## 9. 结论

CScriptEngine 是一个功能完整的 C 语言脚本引擎，采用经典的编译器前端-后端架构，支持 ANTLR4 词法和语法分析。

### 8.1 技术栈

- **前端**：ANTLR4 词法和语法分析
- **中端**：语义分析 + IR 生成（四元式）
- **后端**：X64/X86/RV64 代码生成 + JIT 执行

### 8.2 已实现功能

|------|------|------|
| **数据类型** | int, double, char, bool, float, void | ✅ |
| **构造类型** | 数组、指针、结构体、枚举、typedef | ✅ |
| **运算符** | 算术、关系、逻辑、位运算、赋值 | ✅ |
| **控制流** | if-else, while, for, do-while, switch | ✅ |
| **break/continue** | 循环控制语句 | ✅ |
| **函数** | 定义、调用、递归、参数传递 | ✅ |
| **标准库** | printf, puts, strlen, strcpy, malloc, free | ✅ |
| **代码生成** | x86-64, x86-32, RISC-V 64 | ✅/⚠️ |

### 8.3 待完善功能
