# CScriptEngine AI代码生成规范宪章

## 1. 概述

本文档定义了使用AI辅助开发CScriptEngine项目的规范和最佳实践。通过遵循这些规范，可以确保AI生成的代码符合项目标准，提高开发效率和代码质量。

## 2. AI代码生成流程

### 2.1 设计阶段

1. **架构设计**：使用AI辅助设计系统架构
2. **详细设计**：使用AI生成详细设计文档
3. **API设计**：使用AI设计接口和数据结构

### 2.2 代码生成阶段

1. **模块划分**：按照架构设计划分模块
2. **代码生成**：基于详细设计生成代码
3. **代码审查**：审查AI生成的代码

### 2.3 测试阶段

1. **测试生成**：使用AI生成测试用例
2. **测试执行**：运行测试验证功能
3. **优化**：基于测试结果优化代码

## 3. AI代码生成指令规范

### 3.1 指令格式

```
# 任务描述

## 输入信息
- 项目架构：[架构描述]
- 模块功能：[功能描述]
- 技术要求：[技术规范]
- 代码风格：[编码规范]

## 输出要求
- 代码文件：[文件路径]
- 功能实现：[具体功能]
- 测试覆盖：[测试要求]
- 文档要求：[文档要求]
```

### 3.2 指令示例

```
# 实现X64代码生成器的函数调用处理

## 输入信息
- 项目架构：CScriptEngine采用前端-后端架构，后端负责生成x64机器码
- 模块功能：X64Code类需要实现函数调用的代码生成
- 技术要求：遵循Windows x64调用约定，参数通过RCX、RDX、R8、R9传递
- 代码风格：符合项目编码规范，使用PascalCase命名

## 输出要求
- 代码文件：src/backend/X64Code.cpp
- 功能实现：实现CallStandardLibraryFunction方法，支持printf等函数调用
- 测试覆盖：生成测试用例验证函数调用功能
- 文档要求：添加Doxygen风格的函数注释
```

## 4. 代码质量规范

### 4.1 代码风格

- 遵循项目编码规范（见TECHNICAL_SPECIFICATION.md）
- 使用统一的命名约定
- 保持代码缩进和格式一致
- 添加适当的注释

### 4.2 功能完整性

- 实现所有指定的功能
- 处理所有边界情况
- 正确处理错误情况
- 支持所有指定的输入类型

### 4.3 性能要求

- 代码应高效执行
- 避免不必要的计算
- 合理使用内存
- 优化关键路径

### 4.4 安全性

- 避免内存泄漏
- 防止缓冲区溢出
- 正确处理异常
- 验证输入参数

## 5. AI代码审查规范

### 5.1 审查要点

| 审查项 | 检查内容 | 标准 |
|--------|----------|------|
| 代码风格 | 命名、缩进、格式 | 符合编码规范 |
| 功能正确性 | 实现指定功能 | 功能完整正确 |
| 性能 | 执行效率 | 高效无冗余 |
| 安全性 | 内存安全 | 无安全隐患 |
| 可维护性 | 代码清晰度 | 易于理解和维护 |
| 测试覆盖 | 测试用例 | 覆盖主要功能 |

### 5.2 审查流程

1. **初步审查**：检查代码风格和基本功能
2. **详细审查**：检查逻辑正确性和性能
3. **测试验证**：运行测试确保功能正常
4. **优化**：提出改进建议

### 5.3 审查工具

- **静态分析**：使用静态分析工具检查代码
- **代码风格检查**：使用工具检查编码风格
- **测试覆盖率**：检查测试覆盖情况
- **性能分析**：分析代码性能

## 6. AI辅助开发最佳实践

### 6.1 提示工程

- **明确任务**：清晰描述需要完成的任务
- **提供上下文**：提供项目架构和相关信息
- **设定约束**：指定技术要求和代码风格
- **示例代码**：提供相关的示例代码
- **迭代改进**：通过多轮对话改进结果

### 6.2 代码生成策略

- **模块化生成**：分模块生成代码，避免一次性生成过多代码
- **增量开发**：从简单功能开始，逐步添加复杂功能
- **测试驱动**：先编写测试，再生成实现代码
- **代码复用**：利用现有代码和模式

### 6.3 常见问题处理

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 代码风格不一致 | AI不了解项目规范 | 提供详细的编码规范 |
| 功能不完整 | 指令不够详细 | 提供更具体的功能描述 |
| 性能问题 | AI未考虑性能 | 明确性能要求 |
| 安全性问题 | AI未考虑安全 | 强调安全要求 |
| 测试不足 | AI未生成测试 | 明确测试要求 |

## 7. AI代码生成模板

### 7.1 类实现模板

```cpp
/**
 * @class [ClassName]
 * @brief [类描述]
 */
class [ClassName] {
private:
    // 成员变量
    [Type] m_[MemberName];

public:
    /**
     * @brief 构造函数
     */
    [ClassName]();

    /**
     * @brief 析构函数
     */
    ~[ClassName]();

    /**
     * @brief [方法描述]
     * @param [paramName] [参数描述]
     * @return [返回值描述]
     */
    [ReturnType] [MethodName]([ParamType] [paramName]);
};
```

### 7.2 函数实现模板

```cpp
/**
 * @brief [函数描述]
 * @param [paramName] [参数描述]
 * @return [返回值描述]
 */
[ReturnType] [FunctionName]([ParamType] [paramName]) {
    // 实现代码
    // 错误处理
    // 返回结果
}
```

### 7.3 测试用例模板

```cpp
TEST([TestSuite], [TestName]) {
    // 测试准备
    [Setup code]
    
    // 执行测试
    [Test code]
    
    // 验证结果
    EXPECT_EQ([expected], [actual]);
}
```
