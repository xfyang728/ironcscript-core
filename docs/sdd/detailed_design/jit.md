# JIT执行设计

## 1. 模块概述

JIT（Just-In-Time）执行器是CScriptEngine的后端组件，负责将目标代码加载到内存并执行。它是编译器的第六个阶段，是代码执行的最后环节。

## 2. 设计原理

JIT执行器通过分配可执行内存，将目标代码复制到内存中，然后将内存地址转换为函数指针并执行。它需要考虑内存管理、权限设置、代码执行等因素。

## 3. 数据结构

### 3.1 可执行内存

可执行内存是JIT执行的核心，包括：
- **内存分配**：使用平台特定的API分配可执行内存
- **内存保护**：设置内存保护属性为可执行
- **内存释放**：在执行完成后释放内存

### 3.2 代码缓冲区

代码缓冲区用于存储生成的目标代码：
- **缓冲区管理**：管理代码缓冲区的分配和释放
- **代码复制**：将生成的代码复制到缓冲区

### 3.3 函数指针

函数指针用于执行生成的代码：
- **类型定义**：定义函数指针类型
- **转换**：将内存地址转换为函数指针
- **调用**：通过函数指针执行代码

## 4. 算法流程

1. **初始化**：准备目标代码，设置初始状态
2. **内存分配**：使用平台特定的API分配可执行内存
3. **内存保护**：设置内存保护属性为可执行
4. **代码复制**：将生成的目标代码复制到分配的内存中
5. **函数转换**：将内存地址转换为函数指针
6. **代码执行**：调用函数指针执行代码
7. **内存释放**：释放分配的内存
8. **返回结果**：返回代码执行的结果

## 5. 接口设计

### 5.1 输入接口
- **目标代码**：来自目标代码生成器的输出

### 5.2 输出接口
- **执行结果**：代码执行的结果

### 5.3 错误处理接口
- **执行错误**：当遇到执行错误时，报告错误

## 6. 实现细节

### 6.1 内存管理实现

CScriptEngine 使用抽象的内存管理接口，支持不同平台的内存操作：

- **MemoryManager**：内存管理基类，定义了内存分配、释放和保护的接口
- **WinMemoryManager**：Windows 平台内存管理实现
  - **内存分配**：使用 `VirtualAlloc` 分配可执行内存
  - **内存保护**：使用 `PAGE_EXECUTE_READWRITE` 保护属性
  - **内存释放**：使用 `VirtualFree` 释放内存
- **RV64MemoryManager**：RISC-V 平台内存管理实现
  - **内存分配**：使用 `mmap` 分配可执行内存
  - **内存保护**：设置内存保护属性为可执行
  - **内存释放**：使用 `munmap` 释放内存

### 6.2 平台抽象

为了支持多平台，CScriptEngine 实现了平台抽象层：

- **PlatformAbstraction**：平台抽象基类
- **X64Platform**：x86-64 平台实现
- **RiscV64Platform**：RISC-V 64 平台实现
- **PlatformFactory**：平台工厂，根据目标平台创建相应的平台实例

### 6.3 执行流程

JIT 执行的详细流程如下：

1. **内存分配**：通过平台特定的内存管理器分配可执行内存
2. **代码复制**：将生成的机器码复制到可执行内存
3. **函数转换**：将内存地址转换为函数指针
4. **代码执行**：调用函数指针执行代码
5. **结果获取**：获取函数执行的返回值
6. **内存释放**：释放分配的可执行内存

### 6.4 集成到编译器

JIT执行器与目标代码生成器集成，接收目标代码，执行后返回结果。集成流程如下：

1. **代码生成**：CodeGenerator 生成目标平台的机器码
2. **内存分配**：MemoryManager 分配可执行内存
3. **代码加载**：将机器码复制到可执行内存
4. **代码执行**：调用生成的函数
5. **结果返回**：返回执行结果
6. **内存释放**：释放可执行内存
