# 语义分析器设计

## 1. 模块概述

语义分析器是CScriptEngine的前端组件，负责对抽象语法树进行语义分析，包括符号表管理、类型检查、作用域分析等。它是编译器的第三个阶段，为后续的中间代码生成提供输入。

## 2. 设计原理

语义分析器通过遍历抽象语法树，构建符号表，检查类型兼容性，分析作用域等，确保程序的语义正确性。它是连接前端和后端的桥梁，确保生成的中间代码是语义正确的。

## 3. 数据结构

### 3.1 符号表

符号表是语义分析器的核心数据结构，用于存储程序中的符号信息：
- **符号类型**：变量、函数、类型等
- **符号信息**：名称、类型、作用域、内存偏移量等
- **作用域**：全局作用域、局部作用域（函数体）

### 3.2 类型系统

类型系统定义了程序中使用的数据类型：
- **基本类型**：int、double、char、bool、float、void
- **构造类型**：数组、指针、结构体、联合体、枚举
- **类型兼容性**：定义不同类型之间的兼容性规则

### 3.3 错误信息

错误信息用于报告语义错误：
- **错误类型**：未定义变量、未定义函数、类型不匹配、参数数量不匹配等
- **错误位置**：错误发生的文件、行号、列号
- **错误消息**：描述错误的详细信息

## 4. 算法流程

1. **初始化**：创建全局符号表，准备抽象语法树
2. **符号表构建**：遍历抽象语法树，收集符号信息，构建符号表
3. **类型检查**：检查表达式和语句的类型是否正确
4. **作用域分析**：分析变量和函数的作用域
5. **错误报告**：报告语义错误
6. **输出**：返回经过语义分析的抽象语法树

## 5. 接口设计

### 5.1 输入接口
- **抽象语法树**：来自语法分析器的输出

### 5.2 输出接口
- **经过语义分析的抽象语法树**：作为中间代码生成器的输入

### 5.3 错误处理接口
- **语义错误**：当遇到语义错误时，报告语义错误

## 6. 实现细节

### 6.1 符号表管理

符号表管理的实现文件包括：
- **SymbolTable.h**：符号表和符号的定义
- **SymbolTable.cpp**：符号表的实现

符号表管理功能：
- **符号插入**：将符号添加到符号表中
- **符号查找**：在符号表中查找符号
- **作用域管理**：处理不同作用域的符号
- **内置函数注册**：注册内置函数到符号表

### 6.2 类型检查

类型检查的实现文件包括：
- **SemanticAnalyzer.cpp**：语义分析器的实现
- **SemanticAnalyzer.h**：语义分析器的定义

类型检查功能：
- **表达式类型检查**：检查表达式的类型是否正确
- **赋值类型检查**：检查赋值操作的类型是否兼容
- **函数调用类型检查**：检查函数调用的参数类型是否匹配
- **数组类型检查**：检查数组声明和访问的类型
- **指针类型检查**：检查指针操作的类型
- **结构体类型检查**：检查结构体成员访问的类型

### 6.3 错误处理

错误处理功能：
- **错误检测**：检测语义错误
- **错误报告**：生成错误消息
- **错误恢复**：在遇到错误时尝试继续分析
- **错误计数**：统计错误数量

### 6.4 集成到编译器

语义分析器与语法分析器和中间代码生成器集成，接收抽象语法树，进行语义分析后传递给中间代码生成器。集成流程如下：

1. **语法分析**：Parser 生成 Parse Tree
2. **AST 构建**：ASTBuilder 将 Parse Tree 转换为 AST
3. **语义分析**：SemanticAnalyzer 对 AST 进行语义分析
4. **中间代码生成**：IRGenerator 生成四元式

### 6.5 内置函数支持

语义分析器支持以下内置函数：
- **stdio.h**：printf, scanf, puts, gets, getchar, putchar, fopen, fclose, fread, fwrite, fprintf, fscanf
- **string.h**：strlen, strcpy, strcat, strcmp, strstr, memset, memcpy
- **stdlib.h**：malloc, free, realloc, system, getenv
- **math.h**：sqrt, pow, sin, cos, tan, abs, fabs
- **time.h**：time, clock
